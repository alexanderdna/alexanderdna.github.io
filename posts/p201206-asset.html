Đóng gói và truy cập asset trong Unity

Bài viết dựa trên kinh nghiệm cá nhân của tác giả khi làm việc với Asset Bundles trong một số dự án game. Addressables cũng là một công cụ hữu ích, nhưng sẽ không được đề cập trong bài viết này vì tác giả chưa rành cách xài :>

Giới thiệu

Khi làm một dự án Unity mà số lượng asset tương đối lớn, ta không thể chỉ dùng ScriptableObject hoặc thư mục Resources để quản lý các asset này. May thay, Asset Bundles là một giải pháp rất tốt. Ta có thể phân các asset vào những nhóm nhỏ dựa trên công dụng hoặc nội dung của chúng và cho vào từng bundle riêng. Các bundle này có thể được đóng gói chung với game hoặc để trên server và tải xuống sau ở dạng DLC.

Bài viết này cung cấp một "dàn ý" về cách lưu trữ asset trong các Asset Bundles cũng như cách quản lý và truy cập những asset này trong lúc chạy game.

Chuẩn bị

Để đưa asset vào một bundle, cách thủ công là gán tên bundle cho nó ở cạnh dưới cửa sổ Inspector. Nhưng với số lượng asset khá lớn thì đó không phải cách hay. Thay vào đó, ta dùng AssetBundle Browser, một plugin do chính Unity cung cấp và có thể cài đặt thông qua Package Manager.

Cách sử dụng chi tiết của AssetBundle Browser có thể xem tại link sau:

https://docs.unity3d.com/Manual/AssetBundles-Browser.html

Sau khi tạo các bundle tùy nhu cầu và kéo asset vào xong, chúng ta có thể tiến hành công đoạn đóng gói bằng nút Build trong cửa sổ plugin. Những bundle đã đóng gói sẽ nằm trong thư mục StreamingAssets nếu bạn có đánh dấu vào mục Copy to StreamingAssets.

Trước khi build game, bạn nhớ chạy lệnh Build của AssetBundle Browser để cập nhật đúng và đủ những asset cần sử dụng trong game.

Tải các bundle lên khi chạy game

Một khi đã chuẩn bị xong các asset bundle thì việc tải chúng lên bộ nhớ và truy cập các asset bên trong sẽ không quá phức tạp.

Ta tạo một class quản lý asset như sau:

using System.Collections.Generic;
using UnityEngine;

public sealed class GameAssetManager : MonoBehaviour
{
    public static GameAssetManager Instance { get; private set; }

    public void Awake()
    {
        if (Instance == null)
        {
            Instance = this;
            DontDestroyOnLoad(gameObject);

            // hàm này từ từ sẽ có
            preloadBundles();
        }
        else if (Instance != this)
        {
            Destroy(gameObject);
        }
    }
}

Theo đoạn code trên, bạn có thể thấy chúng ta sẽ dùng một thực thể dạng singleton để quản lý việc tải bundle và truy xuất asset. Bạn hoàn toàn có thể dùng một mô hình khác nếu không thích singleton.

Qua Unity, chờ compile xong code mới, ta tạo một GameObject trong scene khởi động của game (scene đầu tiên mở ra khi chạy game) và thêm component GameAssetManager. Nếu không đụng chạm gì nữa thì GameObject này sẽ tồn tại cho tới khi tắt game.

Nhớ đặt tên cho GameObject đó để dễ phân biệt.

Chúng ta tiếp tục khai báo một danh sách các bundle có trong game:

    private static readonly string[] bundleNames =
    {
        "(common)",
        "(model)(islands)",
        "(model)(people)",
        "(shader)(plant)",
        "(shader)(people)",
        "(ui)(images)",
    };

Theo sau là một bảng chứa các bundle sẽ được nạp:

    private Dictionary<string, AssetBundle> _bundles;

Bạn có thể thấy trong lúc khởi tạo, chúng ta gọi một hàm tên là preloadBundles, và sau đây nội dung của nó:

    private void preloadBundles()
    {
        _bundles = new Dictionary<string, AssetBundle>();

        for (int i = 0, c = bundleNames.Length; i < c; ++i)
        {
            var bundleName = bundleNames[i];

            string bundlePath;

            // Đường dẫn tới các asset bundle sẽ khác nhau
            // tùy theo nền tảng, nên ta chia nhánh code.

#if UNITY_IOS || UNITY_EDITOR
            bundlePath = string.Concat(Application.streamingAssetsPath,
                "/", bundleName);

#elif UNITY_ANDROID
            bundlePath = string.Concat(Application.dataPath,
                "!assets/", bundleName);

#else
            bundlePath = "";
#endif

            var bundle = AssetBundle.LoadFromFile(bundlePath);
            _bundles.Add(bundleName, bundle);
        }
    }

Cuối cùng, để truy xuất một asset nhất định, ta có hàm sau đây:

    public T Load<T>(string bundleName, string name)
        where T : UnityEngine.Object
    {
        T obj;

        var bundle = _bundles[bundleName];
        obj = bundle.LoadAsset<T>(name);

        return obj;
    }

Ví dụ, ta cần load một tấm hình để đưa vào UI Image. Giả sử tấm hình tên là button_yellon và bundle có tên là (ui)(images) thì code sẽ trông từa tựa như sau:

    var sprite = GameAssetManager.Instance.Load<Sprite>(
        "(ui)(images)", "button_yellow");

    myButtonImage.sprite = sprite;
)

Cải tiến

Nội dung của GameAssetManager đang trong giai đoạn thô sơ. Nó có một số hạn chế như sau:

  1. Mỗi lần có thay đổi về asset là phải đóng gói lại mới có thể chạy test, dù chỉ trên editor.
  2. Danh sách bundle đang được để trong code, khó quản lý nếu người làm asset không có thẩm quyền về code.

Nhược điểm số 1 có thể được giải quyết bằng một phiên bản sử dụng AssetDatabase, hỗ trợ truy cập trực tiếp các asset nếu chạy test trên Unity Editor mà không cần đóng gói trước. Phiên bản này được đặt ở cuối bài.

Nhược điểm số 2 có thể được giải quyết bằng cách khai báo một biến dạng SerializedField và để người làm asset tự quyết định số lượng cũng như tên của các bundle cần được tải trong game. Biến này có thể kiểu mảng string, hoặc dẫn tới một ScriptableObject để tăng tính khu biệt. Các bạn sẽ là người viết phiên bản này nhé, cho tay chân vận động một chút.

Ngoài ra, hàm LoadAsset sẽ chạy theo kiểu synchronous. Nghĩa là nếu asset tương đối lớn thì game sẽ bị khựng một vài giây. Bạn có thể nghiên cứu và chuyển qua xài LoadAssetAsync để tối ưu hóa trải nghiệm người dùng.

Source code tham khảo

using System.Collections.Generic;
using UnityEngine;

public sealed class GameAssetManager : MonoBehaviour
{
#if UNITY_EDITOR
    private const bool useAssetDatabase = true;
#endif

    private static readonly string[] bundleNames =
    {
        "(common)",
        "(model)(islands)",
        "(model)(people)",
        "(shader)(plant)",
        "(shader)(people)",
        "(ui)(images)",
    };

    public static GameAssetManager Instance { get; private set; }

    private Dictionary<string, AssetBundle> _bundles;

    public void Awake()
    {
        if (Instance == null)
        {
            Instance = this;
            DontDestroyOnLoad(gameObject);

            preloadBundles();
        }
        else if (Instance != this)
        {
            Destroy(gameObject);
        }
    }

    private void preloadBundles()
    {
#if UNITY_EDITOR
        if (useAssetDatabase)
        {
            return;
        }
#endif

#pragma warning disable CS0162 // Unreachable code detected
        _bundles = new Dictionary<string, AssetBundle>();
#pragma warning restore CS0162 // Unreachable code detected

        for (int i = 0, c = bundleNames.Length; i < c; ++i)
        {
            var bundleName = bundleNames[i];

#pragma warning disable CS0162 // Unreachable code detected
            string bundlePath;
#pragma warning restore CS0162 // Unreachable code detected

#if UNITY_IOS || UNITY_EDITOR
            bundlePath = string.Concat(Application.streamingAssetsPath, "/", bundleName);
#elif UNITY_ANDROID
            bundlePath = string.Concat(Application.dataPath, "!assets/", bundleName);
#else
            bundlePath = "";
#endif

            var bundle = AssetBundle.LoadFromFile(bundlePath);
            _bundles.Add(bundleName, bundle);
        }
    }

    public T Load<T>(string bundleName, string name) where T : UnityEngine.Object
    {
        T obj;

#if UNITY_EDITOR
        if (useAssetDatabase)
        {
            var paths = UnityEditor.AssetDatabase.GetAssetPathsFromAssetBundleAndAssetName(bundleName, name);
            if (paths.Length > 0)
            {
                var path = UnityEditor.AssetDatabase.GetAssetPathsFromAssetBundleAndAssetName(bundleName, name)[0];
                obj = UnityEditor.AssetDatabase.LoadAssetAtPath<T>(path);
            }
            else
            {
                obj = null;
            }
        }
        else
        {
#endif

#pragma warning disable CS0162 // Unreachable code detected
            var bundle = _bundles[bundleName];
            obj = bundle.LoadAsset<T>(name);
#pragma warning restore CS0162 // Unreachable code detected

#if UNITY_EDITOR
        }
#endif

        return obj;
    }
}
